## D3 启用门禁

- Real Adapter 与 Mock Provider 通过同一套 ToolResult 契约测试。
- `trace_id` 可透传到调用日志。
- `NOT_FOUND`、`PARTIAL_SUCCESS`、`TIMEOUT`、`DEGRADED`、`FAILED` 均有明确映射。
- 真实写入类 Provider 必须具备权限、幂等、审计和显式确认门禁。
- 试点清单明确哪些 Provider 使用 Real，哪些继续使用 Mock 或降级。
- 已填写确认单经过接口负责人和研发负责人确认。
- 已保存真实联调记录、契约测试报告、超时/熔断/降级演练结果。
- 已完成灰度启用方案确认和回滚演练确认。
